Boyer-Moore - найшвидший у всіх випадках. Час пошуку від 0.00017 до 0.0004 секунди. Цей алгоритм ефективний тому, що може пропускати великі шматки тексту, коли знаходить невідповідність

Knuth-Morris-Pratt - у 3-4 рази повільніший за Boyer-Moore (0.0007-0.001 секунди). Він не пропускає символи, але й не повертається назад завдяки префікс-функції

Rabin-Karp - найповільніший. У 6-9 разів повільніший за Boyer-Moore (0.0015-0.0028 секунди). Він використовує хешування, що вимагає додаткових обчислень для кожної позиції

Тож для пошуку підрядка Boyer-Moore - найкращий вибір завдяки швидкості, а Rabin-Karp краще використовувати для множинного пошуку патернів одночасно
